# 深入理解计算机系统：主存（Main Memory）与 x86-64 数据传送指令

在计算机科学中，理解主存的工作原理以及 CPU 如何与其交互，是深入掌握系统架构的关键一步。本文基于 x86-64 架构，系统解析主存的结构、数据格式，以及核心汇编指令（mov、push/pop）的使用规则和实战场景。

## 一、主存的物理与逻辑基础
### 1.1 内存的物理实现方式
内存芯片并非采用“线性一维”设计，而是通过**矩阵（Matrix）结构**实现。这种设计避免了制造超长的线性存储单元，大幅提升了空间利用率和数据访问效率。

### 1.2 内存地址的核心意义
- 一个内存地址唯一指向**一个字节（Byte）** 的存储单元；
- 基本单位换算：1 字节 = 8 位（1 byte = 8 bits）。

### 1.3 x86-64 架构的主存特性
64 位（x86-64）系统中，主存的核心特征：
- 理论寻址范围：$2^{64}$ 个独立内存地址；
- 地址表示宽度：每个地址需 64 位（bits）二进制数表示；
- 寻址精度：每个地址精确指向 1 个字节（8 bits），而非更大的单元。

## 二、数据格式与 mov 指令核心规则
mov 是汇编中最基础的**数据传送指令**，仅完成“数据复制”（源操作数 → 目的操作数），不会建立源/目的的持续关联。指令后缀明确操作数的大小，是使用 mov 的核心要点。

### 2.1 mov 指令家族（按操作数大小分类）
| 指令后缀 | 名称               | 操作数大小 | 说明               |
|----------|--------------------|------------|--------------------|
| movb     | move byte          | 8 位       | 移动 1 个字节      |
| movw     | move word          | 16 位      | 移动 1 个字（2 字节） |
| movl     | move double word   | 32 位      | 移动 1 个双字（4 字节） |
| movq     | move quad word     | 64 位      | 移动 1 个四字（8 字节） |
| movabsq  | move absolute quad | 64 位      | 移动绝对 64 位常量 |

### 2.2 操作数的三种核心形式
mov 指令的操作数只能是以下三类之一，决定了数据的来源/去向：
| 操作数类型 | 格式示例       | 说明                          |
|------------|----------------|-------------------------------|
| 立即数     | $5、$0x108     | 常量值，前缀 `$` 标识         |
| 寄存器     | %rax、%rcx     | CPU 内置寄存器中的值          |
| 内存       | (%rax)、0x104  | 通过地址计算得到的内存单元值  |

### 2.3 寻址模式详解（实战示例）
#### 前提条件
- 寄存器状态：%rax = 0x100，%rcx = 0x1，%rdx = 0x3；
- 内存状态：M[0x100] = 0xFF，M[0x104] = 0xAB，M[0x108] = 0x13，M[0x10C] = 0x11。

#### 各类寻址模式计算结果
| 操作数格式       | 计算结果 | 寻址类型         | 计算逻辑                     |
|------------------|----------|------------------|------------------------------|
| %rax             | 0x100    | 寄存器寻址       | 直接读取 %rax 寄存器的值     |
| 0x104            | 0xAB     | 绝对寻址         | 直接读取地址 0x104 的字节值  |
| $0x108           | 0x108    | 立即数寻址       | 常量本身，无需访问内存/寄存器 |
| (%rax)           | 0xFF     | 间接寻址         | %rax 存储的地址 → 0x100 → 取值 0xFF |
| 4(%rax)         | 0xAB     | 基址+偏移        | 0x100 + 4 = 0x104 → 取值 0xAB |
| 9(%rax,%rdx)     | 0x11     | 比例变址         | 0x100 + 0x3 + 9 = 0x10C → 取值 0x11 |
| 260(%rcx,%rdx)   | 0x13     | 比例变址         | 0x1 + 0x3 + 260(0x104) = 0x108 → 取值 0x13 |
| 0xFC(,%rcx,4)    | 0xFF     | 比例变址         | 0x1×4 + 0xFC = 0x100 → 取值 0xFF |
| (%rax,%rdx,4)    | 0x11     | 比例变址         | 0x100 + 0x3×4 = 0x10C → 取值 0x11 |

## 三、高级数据传送：符号扩展与零扩展
当将“小尺寸数据”移入“大尺寸寄存器”时，需明确**高位填充规则**，对应两类专用扩展指令。

### 3.1 零扩展（Zero-extending）
- 规则：高位全部填充 0；
- 指令前缀：movz（如 movzbw、movzbl、movzbq、movzwl、movzwq）；
- 特殊等价：movzlq %ecx, %rax ≡ movl %ecx, %eax（x86-64 中 32 位移动指令会自动将寄存器高 32 位置零）。

### 3.2 符号扩展（Sign-extending）
- 规则：高位填充数据的“符号位”（最高位）；
- 指令前缀：movs（如 movsbw、movsbl、movsbq、movswl、movswq、movslq）；
- 专用指令：cltq ≡ movslq %eax, %rax（仅用于 %eax 扩展到 %rax）。

### 3.3 movabsq 的特殊用途
movq 无法完整传输 64 位立即数（会截断高 32 位），需使用 movabsq：
```asm
# 错误：%rax 仅得到 0x0000000044556677（高 32 位被截断）
movq $0x0011223344556677, %rax

# 正确：%rax 得到完整的 0x0011223344556677
movabsq $0x0011223344556677, %rax
```

## 四、避坑指南：mov 指令常见错误
x86-64 汇编中，mov 指令的错误多源于“操作数尺寸不匹配”“寻址规则违反”，以下是高频错误及原因：

| 错误代码                | 错误原因分析                                  |
|-------------------------|-----------------------------------------------|
| movb $0xF, (%ebx)       | 64 位系统需用 64 位寄存器寻址（如 %rbx），而非 32 位 %ebx |
| movl %rax, (%rsp)       | 后缀 `l`（32 位）与 %rax（64 位寄存器）尺寸不匹配 |
| movw (%rax), 4(%rsp)    | 源/目的操作数不能同时为内存引用（需通过寄存器中转） |
| movb %al, %sl           | 不存在 %sl 寄存器（寄存器名称错误）|
| movq %rax, $0x123       | 立即数不能作为数据写入的目的（仅可作为源）|
| movl %eax, %dx          | %dx（16 位）无法接收 32 位数据（尺寸不匹配）|
| movb %si, 8(%rbp)       | 后缀 `b`（8 位）与 %si（16 位寄存器）尺寸不匹配 |

## 五、实战：C 语言与 x86-64 汇编的映射
### 5.1 指针操作：Swap 函数的汇编实现
#### C 代码（指针交换）
```c
void swap(long *xp, long *yp) {
    long t0 = *xp;  // 读取 xp 指向的内存值
    long t1 = *yp;  // 读取 yp 指向的内存值
    *xp = t1;       // 将 t1 写入 xp 指向的内存
    *yp = t0;       // 将 t0 写入 yp 指向的内存
}
```

#### x86-64 汇编（参数存储规则：xp→%rdi，yp→%rsi）
```asm
swap:
    movq (%rdi), %rax  # t0 = *xp：读取 %rdi 指向的内存值到 %rax
    movq (%rsi), %rdx  # t1 = *yp：读取 %rsi 指向的内存值到 %rdx
    movq %rdx, (%rdi)  # *xp = t1：将 %rdx 写入 %rdi 指向的内存
    movq %rax, (%rsi)  # *yp = t0：将 %rax 写入 %rsi 指向的内存
    ret                # 函数返回
```

### 5.2 栈操作：push / pop 指令的底层逻辑
x86-64 栈的核心特征：**向低地址方向增长**（栈顶指针 %rsp 减小 = 入栈，增大 = 出栈）。

#### 指令等价关系
| 汇编指令 | 等价实现代码 | 说明 |
|----------|--------------|------|
| push %rax | ```asm
subq $8, %rsp      # 栈顶指针向下移动 8 字节（分配空间）
movq %rax, (%rsp)  # 将 %rax 写入新栈顶位置
``` | 入栈：先分配空间，再写入数据 |
| pop %rax | ```asm
movq (%rsp), %rax  # 从栈顶读取数据到 %rax
addq $8, %rsp      # 栈顶指针向上移动 8 字节（释放空间）
``` | 出栈：先读取数据，再释放空间 |

#### 栈操作示例流程
假设初始栈顶指针 %rsp = 0x8000，执行以下指令的变化：
1. `mov $0x123, %rax` → %rax = 0x123；
2. `push %rax` → %rsp = 0x7FF8，M[0x7FF8] = 0x123；
3. `mov $0x22, %rax` → %rax = 0x22；
4. `push %rax` → %rsp = 0x7FF0，M[0x7FF0] = 0x22；
5. `pop %rax` → %rax = 0x22，%rsp = 0x7FF8（栈顶回到 0x7FF8，保留 0x123）。

## 总结
本文核心要点：
1. x86-64 主存以“字节”为最小寻址单元，mov 指令通过后缀（b/w/l/q）明确操作数尺寸，是避免错误的核心；
2. 寻址模式决定了数据的读取方式，需区分寄存器/立即数/内存三类操作数，以及基址+偏移、比例变址等复合寻址；
3. 扩展指令（movz/movs）解决小尺寸数据向大寄存器传输的高位填充问题，push/pop 本质是栈指针操作+mov 指令的组合；
4. C 语言的指针/栈操作可直接映射为 x86-64 汇编，核心是寄存器分配和内存寻址的对应关系。
